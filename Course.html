DBMS Queries

1. What is a Table?

A table in a DBMS is a collection of related data entries that consists of rows and columns. It stores information in structured format.

2. Use of Table

Tables help store, organize, and manipulate data efficiently using SQL commands.

3. How to Create a Table

We use the CREATE TABLE command to define a table schema.

CREATE TABLE students (
  student_id INT PRIMARY KEY,
  name VARCHAR(100),
  age INT,
  department VARCHAR(50)
);

4. Inserting Data into a Table

Use the INSERT INTO command to add data into a table.

INSERT INTO students (student_id, name, age, department)
VALUES (1, 'Aarav Singh', 21, 'Computer Science');

5. Selecting Data from a Table

Use the SELECT command to retrieve data from a table.

SELECT * FROM students;

SELECT name, department FROM students WHERE age > 20;

6. Updating Data in a Table

Use the UPDATE command to modify existing data.

UPDATE students SET age = 22 WHERE student_id = 1;

7. Deleting Rows from a Table

Use the DELETE command to remove records from a table.

DELETE FROM students WHERE student_id = 1;

8. Altering a Table

Use the ALTER TABLE command to modify the structure of a table.

-- Add a new column
ALTER TABLE students ADD email VARCHAR(100);

-- Drop a column
ALTER TABLE students DROP COLUMN age;

-- Rename a column (MySQL syntax)
ALTER TABLE students RENAME COLUMN name TO full_name;

9. Important SQL Clauses (Explained)

SQL clauses are keywords used in SQL queries to define conditions, filters, groupings, and other behavior. Here are the most commonly used SQL clauses:

  • 1. SELECT: Retrieves data from one or more tables.
    SELECT name, age FROM students;
  • 2. FROM: Specifies the table to retrieve data from.
    SELECT * FROM employees;
  • 3. WHERE: Filters records that fulfill a specified condition.
    SELECT * FROM employees WHERE age > 30;
  • 4. GROUP BY: Groups rows that have the same values in specified columns.
    SELECT department, COUNT(*) FROM employees GROUP BY department;
  • 5. HAVING: Filters groups created by GROUP BY.
    SELECT department, COUNT(*) FROM employees 
      GROUP BY department HAVING COUNT(*) > 5;
  • 6. ORDER BY: Sorts the result-set in ascending or descending order.
    SELECT * FROM students ORDER BY name ASC;
  • 7. LIMIT / OFFSET: Limits the number of rows returned by the query (MySQL/PostgreSQL).
    SELECT * FROM students LIMIT 10 OFFSET 5;
  • 8. DISTINCT: Removes duplicate rows from the result.
    SELECT DISTINCT department FROM employees;
  • 9. IN: Used to check if a value exists in a set of values.
    SELECT * FROM employees WHERE department IN ('HR', 'IT');
  • 10. BETWEEN: Filters the result within a specific range.
    SELECT * FROM products WHERE price BETWEEN 100 AND 500;
  • 11. LIKE: Searches for a specified pattern in a column.
    SELECT * FROM users WHERE name LIKE 'A%';
  • 12. IS NULL / IS NOT NULL: Checks for NULL values.
    SELECT * FROM employees WHERE manager_id IS NULL;
  • 13. AS (Alias): Renames a column or table temporarily.
    SELECT name AS student_name FROM students;
  • 14. JOIN Clauses: Used to combine rows from two or more tables:
    • INNER JOIN - Only matching rows
    • LEFT JOIN - All from left + matched from right
    • RIGHT JOIN - All from right + matched from left
    • FULL OUTER JOIN - All matched and unmatched from both
    SELECT employees.name, departments.dept_name
      FROM employees
      INNER JOIN departments ON employees.dept_id = departments.id;

10. Examples for Each SQL Clause

1. SELECT

Retrieve specific columns from a table.

SELECT name, age FROM students;

2. FROM

Defines the source table.

SELECT * FROM students;

3. WHERE

Filters rows based on conditions.

SELECT * FROM students WHERE age > 20;

4. GROUP BY

Groups rows that share a value in a column.

SELECT department, COUNT(*) FROM employees GROUP BY department;

5. HAVING

Filters grouped records.

SELECT department, COUNT(*) FROM employees 
      GROUP BY department 
      HAVING COUNT(*) > 3;

6. ORDER BY

Sorts result set.

SELECT * FROM students ORDER BY age DESC;

7. LIMIT

Limits number of records returned.

SELECT * FROM students LIMIT 5;

8. OFFSET

Skips a number of rows before beginning to return rows.

SELECT * FROM students LIMIT 5 OFFSET 3;

9. DISTINCT

Removes duplicate rows.

SELECT DISTINCT department FROM employees;

10. IN

Checks if value exists in a set.

SELECT * FROM students WHERE class IN ('A', 'B');

11. BETWEEN

Selects within a range.

SELECT * FROM products WHERE price BETWEEN 100 AND 500;

12. LIKE

Searches for a pattern.

SELECT * FROM users WHERE name LIKE 'A%';

13. IS NULL / IS NOT NULL

Checks for NULL values.

SELECT * FROM employees WHERE manager_id IS NULL;

14. AS (Alias)

Renames a column in the result.

SELECT name AS student_name FROM students;

15. INNER JOIN

Returns rows with matching values in both tables.

SELECT s.name, c.course_name 
      FROM students s 
      INNER JOIN courses c ON s.course_id = c.id;

16. LEFT JOIN

Returns all rows from the left table, and matched rows from the right.

SELECT e.name, d.name 
      FROM employees e 
      LEFT JOIN departments d ON e.dept_id = d.id;

17. RIGHT JOIN

Returns all rows from the right table, and matched rows from the left.

SELECT e.name, d.name 
      FROM employees e 
      RIGHT JOIN departments d ON e.dept_id = d.id;

18. FULL OUTER JOIN

Returns all records when there is a match in either table.

SELECT e.name, d.name 
      FROM employees e 
      FULL OUTER JOIN departments d ON e.dept_id = d.id;

19. UNION

Combines result-sets of two SELECT statements (no duplicates).

SELECT name FROM students
      UNION
      SELECT name FROM teachers;

20. UNION ALL

Combines result-sets and includes duplicates.

SELECT name FROM students
      UNION ALL
      SELECT name FROM teachers;
