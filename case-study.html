1 - 1st position

This query calculates the total marks of each student from four subjects and returns the student who scored the highest. It's useful for identifying the class topper based on overall academic performance.

EXPLANATION:
1. SELECT roll_no, name, (dbms + os + spm + pun) AS total_marks
Selects roll number and name of the student.
Calculates total marks by adding scores from DBMS, OS, SPM, and PUN. Gives the total a temporary name: total_marks.
2. FROM students
Indicates the data source is the students table.
3. ORDER BY total_marks DESC
Sorts the students by total marks in descending order (highest first).
4. LIMIT 1
Restricts the result to only the top 1 student — the one with the highest total marks.

2 - 2nd position

This query finds the student who has the second highest total marks by excluding the top scorer using a subquery. It's helpful for generating rank lists or award selections.

EXPLANATION
1. (dbms + os + spm + pun) AS total_marks
Calculates the total marks of each student.
2. WHERE total_marks < (SELECT MAX(...))
Filters out the student who has the highest total marks.
3. Subquery:
SELECT MAX(...) finds the highest total marks (1st position).
4. ORDER BY total_marks DESC
Sorts the remaining students from highest to lowest.
5. LIMIT 1
Returns only the student with the second highest total marks.

3 - 3rd position

This query retrieves the student who secured the third highest total marks by excluding the top two totals using nested subqueries.

Explanation:
1. Inner-most Subquery
Finds the highest total marks (1st position).
2. Middle Subquery
Excludes the top scorer and finds the 2nd highest total.
3. Outer Query
Excludes the top two scores and selects the next highest — the 3rd position.
4. ORDER BY total_marks DESC
Sorts the eligible students from highest to lowest.
5. LIMIT 1
Returns the student with the third highest total.

This query identifies the student who scored the highest marks in the DBMS subject. It's useful for subject-wise toppers and performance analysis.

5 - 2nd position in DBMS

To find the student who scored the second highest marks in DBMS, we use a subquery to first exclude the highest mark, and then find the next maximum. This helps in identifying the runner-up in that subject.

Explanation:
1. SELECT MAX(dbms) FROM students
Finds the highest DBMS mark in the class.
2. WHERE dbms < (highest mark)
Filters out the top scorer(s).
3. MAX(dbms) from the filtered list
Finds the second highest mark.
4. Main Query
Selects the student(s) who scored exactly that second highest mark in DBMS.

6 - 3rd position in DBMS

To get the 3rd highest marks in the DBMS subject, we use nested subqueries to remove the top two highest marks and then find the next maximum. This allows us to accurately identify the student ranked third in DBMS.

Explanation:
1. Innermost subquery:
SELECT MAX(dbms) FROM students
Finds the highest DBMS mark.
2. Second subquery:
SELECT MAX(dbms) FROM students WHERE dbms < (1st highest)
Finds the 2nd highest DBMS mark.
3. Outer subquery:
SELECT MAX(dbms) FROM students WHERE dbms < (2nd highest)
Finds the 3rd highest DBMS mark.
4. Main query:
Returns the roll_no, name, and dbms mark of the student(s) with that 3rd highest score.

2nd Report

The SQL query is used to find the student(s) who got the same total marks in the marks table.

1. Why Use (dbms + os + spm + pun) Instead of a Separate Column?

You're calculating total marks by summing subject columns

This method is:
Dynamic: You don’t need to create a new column in the table.
Efficient for Ad Hoc Queries: Great for quick reports or one-time queries.
But if used frequently, it's better to store total_marks as a computed or stored column to avoid recalculating

2. How Does SQL Execute This Query Internally?

2.second subquery filters out the highest and gets second highest.
3.Third subquery filters out the top two and gets the third highest.
4.Final outer query compares each student's total with the third highest:

5.Returns all students with that total, including if there’s a tie.
